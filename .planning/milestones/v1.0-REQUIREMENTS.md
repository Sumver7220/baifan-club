## v1 Requirements

### Build & Refactor

- [x] **BUILD-01**: 引入 Vite 5.x 取代 VS Code Live Sass，提供 `npm run dev` / `npm run build` 可重現的 build 流程
  - **Outcome:** ✓ Validated — Vite build skeleton established with HMR, hashed assets, PostCSS autoprefixer
- [x] **BUILD-02**: 全面遷移 SCSS `@import` → `@use`（整批完成，消除模組衝突與重複規則）
  - **Outcome:** ✓ Validated — `css/main.scss` single entrypoint, all partials via `@use`
- [x] **BUILD-03**: 消除硬編碼頁數（`css/pages.scss` L15/L34、`js/nav.js` L97/L100），改為從 DOM 動態推導 `.page` 數量
  - **Outcome:** ✓ Validated — `--page-count` derived from live DOM on startup
- [x] **BUILD-04**: Clerk Modal handler 從 `globalThis.openClerkModal` / `closeClerkModal` 遷移至 delegated event listeners，移除 inline `onclick`
  - **Outcome:** ✓ Validated — delegated document-level event handlers, no inline onclick

### RWD

- [x] **RWD-01**: 所有字體大小從固定 `px` 改為 `clamp(min, fluid, max)`，覆蓋 320px–2560px（4K）流暢縮放，無斷點跳躍
  - **Outcome:** ⚠ Partially Validated — desktop (≥768px) passes; mobile (≤767px) has architectural gap (`--font-hero` advisory warning); mobile deferred to next Phase
- [x] **RWD-02**: 全高度值從 `100vh` 改用 `100dvh` + `100vh` fallback，修正 iOS Safari address bar 導致的版面位移
  - **Outcome:** ⚠ Partially Validated — `100dvh` fallback implemented; mobile layout contract (overflow:hidden + full-screen sections) systemically broken on mobile; mobile deferred to next Phase
- [x] **RWD-03**: 主要圖片（hero、clerk 卡片、menu）加入 `aspect-ratio`，並依內容採用 `object-fit: cover/contain`，防止各斷點比例變形
  - **Outcome:** ⚠ Partially Validated — desktop (≥768px) passes; mobile 320px grid collapse (11px/col) deferred to next Phase
