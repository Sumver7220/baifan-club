# Milestone v1.0: RWD 重構 + Vite 升級

**Status:** ✅ SHIPPED 2026-04-19
**Phases:** 1–2
**Total Plans:** 7

## Overview

引入 Vite 建置流程、SCSS 模組化、動態頁數推導、Clerk Modal 重構，並全面應用 clamp() 流體字型、100dvh fallback、圖片 aspect-ratio，使白飯俱樂部官網在桌機（768px–2560px）達到完整 RWD 響應。

**Requirements:** BUILD-01, BUILD-02, BUILD-03, BUILD-04

- [x] 01-03-PLAN.md — 動態頁數與 Clerk Modal delegated listeners（BUILD-03, BUILD-04）

1. ✓ `npm run dev` 啟動 HMR 開發伺服器，SCSS 改動自動刷新

**Depends on:** Phase 1

**Requirements:** RWD-01, RWD-02, RWD-03

**Plans:** 4 plans

Plans:

- [x] 02-02-PLAN.md — 高度現代化：100dvh + fallback，iOS Safari 位移修正（RWD-02）
- [x] 02-03-PLAN.md — 圖片比例防護：cover 圖片比例約束 + menu 安全 contain 策略（RWD-03）

## Milestone Summary

**Key Decisions:**

- Vite 取代 VS Code Live Sass Compiler — 移除 IDE 依賴，提供 HMR + 雜湊 asset 輸出
- SCSS `@use` 模組化 — 消除命名衝突，建立 `css/main.scss` 單一入口
- 動態頁數從 DOM 推導 — 未來新增頁面只需新增 `.page` 元素，無需改 JS
- `clamp()` 流體字型系統 — 純 CSS 解法，320px–2560px 零斷點跳躍
- `100dvh` + `100vh` fallback — 在 iOS Safari 保持版面穩定
- 行動版延後獨立 Phase — 整體架構需系統性重設計，非個別 bug 修補

**Issues Resolved:**
- VS Code Live Sass 依賴移除 ✓
- 頁數硬編碼消除 ✓
- Clerk Modal 全域 handler 解耦 ✓
- 桌機 4K（2560px）字體過小問題修正 ✓
- iOS Safari address bar 位移修正 ✓

**Issues Deferred:**
- 行動版布局（≤767px）— 系統性架構缺口，移交新 Phase
- 效能優化（圖片 lazy-load、字型優化）— 延後至 v2
- Page 7（斗內香檳王）內容 — 待與業主討論
- History API + 錨點深度連結 — 延後至 v2

**Technical Debt Incurred:**
- `--font-hero` 尚未切換為流體 variant（advisory warning，行動版 Phase 時一併評估）
- 行動版 `overflow:hidden` + 全螢幕 section 合約在手機上系統性失效，需新 Phase 完整重設計
