<task type="auto">
  <name>Task 1: Systematic RWD verification at three breakpoints</name>
  <files></files>
  <action>
Execute verification protocol with systematic testing across 320px, 1920px, 2560px:

1. **Setup:**
   - Run `npm run dev` to start HMR server (wait for "ready in XXms")
   - Open browser (Chrome recommended for DevTools)
   - Open DevTools: F12
   - Open Responsive Design Mode: Ctrl+Shift+M

2. **For Each Viewport Size (320px, 1920px, 2560px):**

   a. **Set viewport size:**
      - Type size in DevTools responsive mode or use preset sizes
      - Example: Type "320" in width field, press Enter
      - Allow page to stabilize (~1 second)

   b. **For Each Page (0–8):**
      - Click bottom navigation to navigate to page
      - Visually inspect:
        - **Typography:** Text appears readable, correct size for breakpoint
        - **Layout:** Content fits within viewport
        - **Images:** No distortion, proper aspect ratio
        - **No vertical scroll:** Confirm `overflow: auto` or `hidden` on body (no scrollbar should appear)
      - Note any visual issues on verification checklist

   c. **Spot Check for Scroll Issues:**
      - Open DevTools → Elements → Select `body`
      - Computed styles should show `overflow: hidden`
      - If any page shows vertical scrollbar: **FLAG AS ISSUE**

3. **Typography Verification (RWD-01):**
   - At 320px: Text should be smaller but readable (min clamp values)
   - At 1920px: Text should match current comfortable baseline (mobile testing size)
   - At 2560px: Text should be noticeably larger (max clamp values)
   - **Test:** Visually compare 320px and 2560px; 2560px should be ~50% larger
   - **Formula check:** Use DevTools Computed styles to see calculated font-size

4. **Height Verification (RWD-02):**
   - At 320px: Simulate address bar toggle (scroll to top/bottom to show/hide address bar)
   - **Issue:** If page jumps or repositions when address bar appears/disappears → **FLAG**
   - At 1920px & 2560px: No address bar to simulate, but check `height: 100dvh` is in viewport element

5. **Image Aspect Ratio Verification (RWD-03):**
   - Page 0: Hero should maintain widescreen aspect ratio (no stretching)
   - Page 3: Clerk cards should appear as perfect squares across all breakpoints
   - Page 6: Gallery tiles should maintain aspect ratio (no distortion)
   - Page 8: Environment cards should maintain consistent proportions
   - **Test:** Resize viewport while on same page; images should scale without changing aspect ratio

6. **Touch Target Verification (Criterion 4):**
   - Page 0: "進入" button (home entry CTA)
   - Page 3: All clerk cards and modal buttons
   - Page 6: Gallery cards
   - Page 8: Environment cards
   - Use DevTools to measure button dimensions:
     - Right-click element → Inspect
     - Check box model in DevTools
     - Minimum: 44px × 48px (accessibility standard)
   - **Issue:** If any button < 44×48px → **FLAG**

7. **Record Findings:**
   - Create checklist format (see VERIFICATION.md structure below)
   - Mark ✓ for passed, ✗ for failed, ? for unclear
   - Note specific pages/elements with issues

8. **Compile & Auto-Run:**
   - `npm run build` should pass (confirms no CSS errors from Plans 01-03)
   - If build fails: **BLOCKER**

Detailed procedure will be recorded in 02-04-VERIFICATION.md (created during execution).
