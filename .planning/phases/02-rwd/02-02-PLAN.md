<objective>
Update all height declarations from fixed `100vh` to modern `100dvh` (dynamic viewport height) with `100vh` fallback for older browsers, eliminating iOS Safari address bar layout shifts.

Purpose: Fix iOS Safari layout instability where the address bar appears/disappears, causing visible page jumps that degrade UX on mobile.

Output: Correct height declarations across base, pages, and navigation layers that remain stable on iOS Safari and other mobile browsers.
</objective>

- css/base.scss html/body rules need review for 100vh vs 100dvh usage
- css/nav.scss likely needs similar audit

**Dynamic Viewport Height (dvh) Explanation:**
- `100vh` = full viewport height (affected by browser UI like address bar on mobile)
- `100dvh` = full "dynamic" viewport height (excludes browser UI, more stable)
- Fallback pattern: `height: 100vh; height: 100dvh;` ensures older browsers fall back to 100vh

**Target Implementation:**
For any element expected to fill the screen:
```css
height: 100vh;    /* fallback for older browsers */
height: 100dvh;   /* modern: excludes address bar on mobile */
```

**Scope:**
- Review and document current dvh usage
- Add fallback declarations where missing
- Verify iOS Safari behavior (documented in SUCCESS_CRITERIA)
</context>
