<objective>
Replace all hardcoded font-size values with fluid `clamp()` formulas that scale smoothly across 320px–2560px viewports.

Purpose: Eliminate fixed-pixel typography that appears too small on 4K and ensure consistent text rendering across all device sizes.

Output: Complete typography system with centralized fluid font variables in base.scss and all page files referencing those variables.
</objective>

<context>
@.planning/codebase/ARCHITECTURE.md

**Current State:**
- base.scss already defines some fluid variables: `--font-entry-fluid`, `--font-lead-fluid`, `--font-placeholder-fluid`
- Many pages still have hardcoded px/rem values: page-1 (1.25rem, 1.5rem), page-2 (1.5rem, 1rem), page-3-modal (1.8rem, 2rem, 1.5rem), page-4 (1.5rem), page-5 (0.8rem, 1.5rem, 1.1rem), page-6 (1.5rem, 1rem), page-8 (0.88rem)

**Fluid Typography Strategy:**
- Use `clamp(min-px, fluid-vw, max-px)` formula
- min: Size at 320px viewport
- fluid: Percentage of viewport width (typically 0.8–4.6vw for various scales)
- max: Size at 2560px viewport (4K baseline)
- Example: Large heading: `clamp(1.5rem, 3vw, 3rem)` scales from 1.5rem to 3rem smoothly

**Baseline Sizes at Key Breakpoints (for formula derivation):**
- 320px (mobile): Scale headings down by ~60%, body text by ~90%
- 1920px (desktop): Current "comfortable" baseline (existing design reference)
- 2560px (4K): Scale up by ~15–20%
</context>
