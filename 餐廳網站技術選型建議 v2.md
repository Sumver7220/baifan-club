# 餐廳官方網站專案開發計畫書 (Product Development Plan)
**版本：v2.0**

---

## 1. 專案概述 (Project Overview)

本專案旨在為餐廳業主開發高度還原設計稿、具備沉浸式體驗的官方網站。網站採用單頁左右切換的互動模式，以純手工 HTML/CSS/JS 打造，確保視覺效果與業主提供之設計稿完全一致。考量業主零技術背景，內容更新初期採人工維護方式，待正式上線營運後，視實際需求評估是否導入 Headless CMS 進行局部管理。

---

## 2. 核心目標 (Core Objectives)

- **100% 設計還原**：以肉眼對稿方式，完整還原業主提供之 Canva 設計截圖，包含背景圖、裝飾素材、字型風格與色彩。
- **沉浸式互動體驗**：採左右頁面切換模式，無垂直滾動，提供如翻閱精裝書般的瀏覽體驗。
- **零常態維運成本**：部署於 Cloudflare Pages 免費方案，無伺服器月費，合法支援商業用途。
- **基礎 SEO**：設定完整 `<title>`、`<meta description>` 及 Open Graph 標籤，確保搜尋引擎可索引、社群分享有預覽圖。

---

## 3. 技術選型 (Technology Stack)

| 項目 | 選擇 | 說明 |
|------|------|------|
| 前端技術 | 純 HTML / CSS / JS | 無框架，最大化設計還原自由度 |
| 部署平台 | Cloudflare Pages | 免費、支援商業用途、全球 CDN |
| 版本控制 | GitHub | 搭配 Cloudflare Pages 自動部署 |
| CMS（待定） | Sanity（Free Tier） | 視上線後實際需求，局部導入 |

> **關於 CMS 的決策原則**：內容與版面在此設計中高度耦合，難以事先判斷哪些區塊真正需要業主自行更新。建議待業主實際使用一段時間後，再針對最常變動的區塊（如菜單、店員介紹、香檳王排行）補接 Sanity，避免做白工。

---

## 4. 網站結構 (Site Structure)

網站共 9 個頁面，透過底部固定導覽列左右切換，無垂直滾動。

| 頁面 | 主要內容 |
|------|---------|
| 入口 | 全版背景圖、品牌標題、進入引導文字 |
| 關於 | 左圖右文，品牌介紹與氛圍描述 |
| 顧客守則 | 半透明卡片疊加背景，條列式規範 |
| 店員介紹 | 格狀頭像排列，支援 Coming soon 佔位；點擊頭像開啟 Lightbox，顯示放大圖片與詳細介紹 |
| 服務內容 | 左側文字說明、右側圖片排列 |
| 菜單 | 多張菜單卡片並排展示；點擊卡片開啟 Lightbox，可透過左右箭頭（桌機）或左右滑動（手機）切換上下張 |
| 精彩瞬間 | 六宮格相片牆 |
| 斗內香檳王 | 客製化排行榜版面，含前三名特殊樣式 |
| 店內環境 | 六宮格場地照，附區域名稱標注 |

---

## 5. 互動規格 (Interaction Spec)

- **切換方式**：點擊底部導覽列項目切換頁面
- **切換動畫**：待與業主確認（淡入淡出 / 滑動 / 直接切換）
- **導覽列**：固定於畫面底部，當前頁面高亮顯示
- **RWD**：支援桌機與手機，手機端需另行規劃導覽列收合方式
- **店員介紹 Lightbox**：點擊任一頭像，以覆蓋層方式顯示該員工放大圖片與詳細介紹；點擊遮罩或關閉按鈕可收起
- **菜單 Lightbox**：點擊任一菜單卡片開啟全屏瀏覽模式；桌機以左右箭頭切換張數，手機支援左右觸控滑動手勢切換；顯示當前張數（例：2 / 5）

## 7. 開發階段規劃 (Development Phases)

### 第一階段：素材整理與環境建置
- 建立 GitHub Repo 與 Cloudflare Pages 部署環境
- 整理業主提供之所有素材，建立資料夾結構
- 確認字型來源（Google Fonts 或其他）
- 確認頁面切換動畫方式

### 第二階段：基礎框架建立（UI-First）
- 建立 HTML 基礎結構：固定底部導覽列 + 頁面容器
- 實作左右切換邏輯（JS）
- 實作切換動畫
- 確保 RWD 基礎結構正確

### 第三階段：逐頁切版
- 依序完成 9 個頁面的視覺還原
- 順序建議：入口 → 關於 → 顧客守則 → 店員介紹 → 服務內容 → 菜單 → 精彩瞬間 → 斗內香檳王 → 店內環境
- 每頁完成後部署至 Cloudflare Pages，供業主即時確認

### 第四階段：細節調整與測試
- 跨瀏覽器測試（Chrome / Safari / Firefox）
- 手機裝置測試
- 效能優化（圖片壓縮、載入速度）
- 設定 SEO Meta 標籤與 Open Graph

### 第五階段：正式上線
- 將專屬網域指向 Cloudflare Pages
- 業主最終確認
- 正式上線

### 第六階段（待定）：CMS 局部導入
- 視業主上線後實際需求評估
- 針對最常變動的區塊補接 Sanity
- 預期候選區塊：菜單、店員介紹、斗內香檳王排行

## 9. 風險與預期管理 (Risk & Expectation Management)

- **素材不完整**：業主僅有 Canva 截圖，無完整設計規格。字型、色碼、間距均需自行判斷，可能有細節落差，需與業主溝通「最大程度還原」而非「像素完美」。
- **設計稿彈性限制**：版面由開發者手工刻製，日後若需變更整體排版，需由開發者介入修改程式碼。
- **手機體驗需額外設計**：左右切換在手機上的觸控體驗需特別處理，原設計稿為桌機版，手機版佈局可能需要調整。
- **CMS 時程未定**：第六階段是否執行、執行時程，視業主需求而定，不列入初期報價範圍。
